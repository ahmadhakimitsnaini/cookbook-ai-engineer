# Document Copilot Implementation Checklist

This document tracks the detailed, step-by-step implementation plan for Document Copilot, adhering strictly to the architecture constraints and agent guidelines.

## Phase 1: Project Scaffolding

- [ ] **Backend Setup**
  - [ ] Initialize FastAPI app layout (`app/main.py`, `app/config.py`, `app/api/`, etc.).
  - [x] Add `pydantic-settings` and create `app/config.py` as the single source of truth for environment variables.
  - [ ] Set up `structlog` for structured logging.
  - [x] Initialize `alembic` (`alembic init alembic`) with `alembic.ini` and `alembic/env.py` adjusted for async/SQLAlchemy.
  - [ ] Set up basic health check route to ensure FastAPI runs correctly.
- [ ] **Frontend Setup**
  - [ ] Scaffold Vite + React SPA + TypeScript (do not use Next.js).
  - [ ] Ensure `pnpm` is used as the strict package manager (`.npmrc` is already present).
  - [ ] Configure Tailwind CSS and add global tokens to `src/index.css`.
  - [ ] Setup `src/lib/env.ts` to validate required `VITE_` environment variables.
  - [ ] Install `shadcn/ui` base primitives (e.g., button, input) into `src/components/ui`.
  - [ ] Setup React Router (`src/App.tsx`) with a basic chat route.

## Phase 2: Database and Migrations

- [x] **SQLAlchemy Models**
  - [x] Create `app/database/models.py`.
  - [x] Define `profiles` table (tied to Supabase auth user IDs).
  - [x] Define `chat_threads` and `chat_messages` tables.
  - [x] Define `message_citations` table for grounding answers.
  - [x] Define `source_documents` (metadata, full markdown) and `document_chunks` (text, embedding vector, tsvector for full-text).
- [x] **Alembic Initial Migration**
  - [x] Generate autogenerated base schema migration.
  - [x] Manually modify the migration to include Postgres/Supabase-specific features:
    - [x] `CREATE EXTENSION IF NOT EXISTS vector;`
    - [x] Define HNSW indexes for the embedding vector search.
    - [x] Define GIN indexes for Postgres full-text search (`tsvector`) and JSON metadata.
    - [x] Setup Row-Level Security (RLS) policies linking records to `profiles.id`.
  - [x] Apply migration to local/hosted Supabase DB.

## Phase 3: Auth & Shared Communication Layer

- [ ] **Frontend Auth**
  - [ ] Install `@supabase/supabase-js`.
  - [ ] Create `src/lib/supabase.ts` for browser Supabase client.
  - [ ] Create a login page supporting Email/Password sign-in.
  - [ ] Store user session context in React state.
- [ ] **Backend Token Verification**
  - [ ] Create `app/auth/dependencies.py` to parse `Authorization: Bearer <token>`.
  - [ ] Validate the JWT using Supabase Auth (or via validated signing keys).
  - [ ] Create a `get_current_user` FastAPI dependency.
- [ ] **Frontend API Client**
  - [ ] Create `src/lib/http.ts` wrapping the native `fetch` API.
  - [ ] Ensure automatic injection of the Supabase Bearer token for every authenticated request.
  - [ ] Setup robust error handling (`ApiError`) and timeout logic.

## Phase 4: Core Chat Interface (Stubbed)

- [ ] **Backend Streaming Endpoint**
  - [ ] Create `POST /chat/stream` in `app/api/chat.py`.
  - [ ] Require `get_current_user` dependency.
  - [ ] Write stubbed logic to emit AI SDK-compatible streaming events (delays, text deltas).
- [ ] **Frontend Chat UI**
  - [ ] Add Vercel AI SDK React primitives.
  - [ ] Build the main chat UI (`src/pages/chat/index.tsx` and `src/components/chat/`).
  - [ ] Connect the AI SDK `useChat` hook to point to the FastAPI `/chat/stream` endpoint with the correct transport headers.

## Phase 5: Ingestion Pipeline

- [ ] **Data Processing**
  - [ ] Create one-off `ingest/` scripts to read downloaded HTML/txt SEC filings.
  - [ ] Convert 10-K/10-Q filings to normalized Markdown.
  - [ ] Chunk documents semantically (e.g., by header/section) retaining metadata (year, ticker, section, etc.).
- [ ] **Embeddings & Persistence**
  - [ ] Embed chunks using OpenAI embeddings model via OpenAI SDK.
  - [ ] Write source documents and chunks into Supabase Postgres using the service-role key.

## Phase 6: Hybrid Retrieval Layer

- [ ] **Database Helpers**
  - [ ] Implement `pgvector` semantic search query in `app/retrieval/queries.py`.
  - [ ] Implement Postgres full-text (`to_tsquery`) search in `app/retrieval/queries.py`.
- [ ] **Reciprocal Rank Fusion (RRF)**
  - [ ] Create `app/retrieval/fusion.py`.
  - [ ] Combine semantic and lexical search results to surface the most relevant chunks.
- [ ] **Retriever Module**
  - [ ] Build `DocumentRetriever` class that accepts a user query and returns fused `SourcePassage` records.

## Phase 7: LLM Agent & Grounding

- [ ] **PydanticAI Setup**
  - [ ] Create `app/assistant/agent.py` defining the PydanticAI `Agent`.
  - [ ] Inject `DocumentAgentDeps` (retriever, grounding validator).
  - [ ] Define precise product instructions emphasizing citations and zero hallucinations.
  - [ ] Define the output schema `GroundedAnswer` returning answer text + citation array.
- [ ] **Grounding Enforcement**
  - [ ] Build `app/grounding/validator.py`.
  - [ ] Ensure all generated citations map strictly to the retrieved `SourcePassage` IDs.
  - [ ] Throw controlled validation errors if the agent hallucinates citations.
- [ ] **Integration**
  - [ ] Update `POST /chat/stream` to invoke the PydanticAI agent instead of the stub.
  - [ ] Stream the chunks back to the AI SDK as tool calls or structured annotations.
  - [ ] Persist the final user message, assistant message, and citations into the DB.

## Phase 8: Frontend Polish & Verification

- [ ] **Citation UI**
  - [ ] Build components to render in-line citations `[1]`, `[2]`.
  - [ ] Build a side-panel or pop-up to show the underlying `SourcePassage` excerpt that the agent cited.
- [ ] **Empty States & Errors**
  - [ ] Handle "401 Unauthorized" gracefully (redirect to login).
  - [ ] Handle grounding validation failure gracefully (e.g., "Could not generate a verified answer from the corpus.").
- [ ] **Pilot Testing**
  - [ ] Walk through the 10 example analyst questions from `docs/client-brief.md`.
  - [ ] Ensure exact document links and page numbers are accurately cited.
